# F-020 — Add .fleet/templates/card.md.tmpl so fleet-new-card.sh works for this repo

## Context
`agentic-fleet/scripts/fleet-new-card.sh` was recently fixed to look for its card
template at `<repo>/.fleet/templates/card.md.tmpl` in the target repo (previously it
incorrectly looked for `<repo>/templates/card.md.tmpl`). printer-dashboard did not
have this file at all, so the script would still fail here even after that fix —
this gap was noted while reviewing `agentic-fleet/fleet-index.md`. This card closes
that gap by adding the template file to this repo and verifying the script works
end-to-end against it.

## Plan
1. [x] Implementer: create `.fleet/templates/card.md.tmpl` in this repo, matching the
   frontmatter/body schema `fleet-new-card.sh` substitutes into (`id: K-000`,
   `title: <short title>`, `created: <YYYY-MM-DD>`, `# K-000 — <short title>`).
2. [x] Implementer: run `fleet-new-card.sh` end-to-end against this repo with a
   throwaway scratch card id, confirm it succeeds with no template-path error, then
   delete the throwaway card file.

## Working context
- Template path expected by the script: `.fleet/templates/card.md.tmpl` (repo-root
  relative).
- Canonical content already existed at
  `agentic-fleet/.fleet/templates/card.md.tmpl` (agentic-fleet's own copy of the
  same template) — used as the source of truth for this repo's copy.
- Script: `agentic-fleet/scripts/fleet-new-card.sh`.

## Decision log
- 2026-07-13: Copied the canonical card template verbatim from
  `agentic-fleet/.fleet/templates/card.md.tmpl` into
  `printer-dashboard/.fleet/templates/card.md.tmpl`, since its frontmatter
  lines/placeholders exactly match every `sed` substitution the script performs
  (`id: K-000`, `title: <short title>`, `created: <YYYY-MM-DD>`, and the
  `# K-000 — <short title>` heading). No agentic-fleet-provided generic template
  existed elsewhere under `agentic-fleet/templates/`, so this was the right source
  to copy rather than write from scratch.
- 2026-07-13: Verified end-to-end by running
  `fleet-new-card.sh <printer-dashboard-repo> backlog F-020-TEST "Scratch test card"`
  — it created `.fleet/board/backlog/F-020-TEST.md` successfully with
  id/title/created/heading correctly substituted, with no template-path error. The
  throwaway file was deleted immediately after confirming its contents.
